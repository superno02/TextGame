# Implementation Playbook

## Workflow

1. Confirm the user-facing outcome.
2. Inspect the real code paths before changing files.
3. Make the smallest coherent change set.
4. Validate only the behavior you changed, then summarize risk.

## Output

- Lead with the concrete change or blocking issue.
- Call out assumptions that could invalidate the implementation.
- Keep follow-up suggestions separate from the completed work.
