# 戰鬥系統

本文件說明 TextGame 的戰鬥系統設計，包含戰鬥流程、公式、技能經驗觸發，以及數值平衡參考。

> 最後更新：2026-03-27（從 CombatStats.md 分拆）

---

## 一、戰鬥系統

戰鬥系統已於 `GameEngine` 中實作完成，採用**回合制自動戰鬥**機制。

### 戰鬥流程概述

```
玩家選擇怪物目標 → attackMonster()
  → 建立 CombatMonster（運行時狀態）
  → isInCombat = true
  → Task { @MainActor } → runCombatLoop()
    → 每回合（延遲 0.8 秒產生文字效果）：
      1. 玩家攻擊（命中判定 → 傷害計算）
      2. 怪物死亡判定 → 勝利處理（經驗值 + 升級判定 + 掉落物）
      3. 怪物攻擊（閃避判定 → 傷害計算）
      4. 玩家死亡判定 → 死亡處理（傳送村莊 + 恢復半血）
      5. 吸收回合中觸發的技能經驗
```

### 核心資料結構

| 結構 | 定義位置 | 說明 |
|------|----------|------|
| `CombatMonster` | `GameEngine.swift` | 戰鬥中怪物的運行時狀態，包裝 `MonsterTemplate` 並追蹤可變 `currentHealth` |
| `RoundResult` | `GameEngine.swift` | 回合結果列舉：`continues` / `monsterDefeated` / `playerDefeated` |
| `CombatCalculator` | `GameEngine.swift` | 戰鬥數值計算純函數集（`enum`，無實例化），方便單元測試 |

### 角色戰鬥參數

| 參數 | 來源 | 說明 |
|------|------|------|
| 總攻擊力 `totalAttackPower` | `strength` + 武器 `attackPower` | 物理攻擊力 |
| 總防禦力 `totalDefensePower` | Σ(已裝備防具的 `defensePower`) | 傷害減免 |
| 武器技能 `weaponSkillType` | `SkillType.weaponSkillType(for: itemId)` | 裝備武器對應的技能類型 |
| 閃避技能等級 | `skill(for: .evasion)?.rank` | 閃避機率計算因子 |

### 戰鬥公式

#### 命中率（`calculateHitChance`）

```
命中率 = 0.7 + 敏捷 × 0.01 + 武器技能等級 × 0.02 - 怪物等級 × 0.03
上限 0.95 / 下限 0.30
```

#### 閃避率（`calculateDodgeChance`）

```
閃避率 = 0.05 + 敏捷 × 0.005 + 閃避技能等級 × 0.02 - 怪物等級 × 0.01
上限 0.50 / 下限 0.02
```

#### 傷害計算（`calculateBaseDamage` + `applyRandomVariance`）

```
基礎傷害 = max(攻擊力 - 防禦力, 1)
實際傷害 = max(基礎傷害 × random(0.8 ~ 1.2), 1)
```

### 戰鬥觸發的技能經驗

| 事件 | 觸發技能 | 經驗量 |
|------|----------|--------|
| 攻擊命中 | 武器技能（依裝備武器） | `1.0 + 怪物等級 × 0.5` |
| 攻擊未命中 | 武器技能（依裝備武器） | `0.5` |
| 閃避成功 | 閃避 | `1.5 + 怪物等級 × 0.5` |
| 被擊中（穿甲） | 防具技能（每件裝備各觸發一次） | `0.5 + 怪物等級 × 0.3` |

### 武器 → 技能類型映射

透過 `SkillType.weaponSkillType(for:)` 靜態方法：

| 物品 ID | 武器名稱 | 對應技能 |
|---------|----------|----------|
| `01_01_iron_sword` | 鐵劍 | 劍術 |
| `01_12_warrior_greatsword` | 戰士巨劍 | 劍術 |
| `01_03_short_bow` | 短弓 | 弓術 |
| `01_02_wooden_staff` | 木杖 | 杖術 |
| `01_04_rusty_dagger` | 生鏽匕首 | 匕首 |
| `01_17_long_spear` | 長矛 | 劍術 |
| `01_18_shadow_blade` | 暗影之刃 | 匕首 |
| 其他 / 空手 | — | 無（不觸發武器技能經驗） |

### 防具 → 技能類型推斷

| 條件 | 對應技能 |
|------|----------|
| 裝備欄位為副手（`offHand`） | 盾牌 |
| 防禦力 ≥ 5 | 重甲 |
| 防禦力 < 5 | 輕甲 |

### 勝利處理（經驗值授予）

怪物死亡後依序處理：
1. 授予角色經驗值（`monster.template.experience`）
2. 呼叫 `character.gainExperience(exp)`，若達到升級門檻則自動升級
3. 升級時：屬性依職業 `CircleGrowth` 成長、狀態值重算、HP/MP/SP 全回復
4. 輸出升級訊息與屬性成長明細

### 掉落物處理

怪物死亡後透過 `lootTableId` 查詢掉落表（`LootTableLoader`）：
1. 若 `lootTableId` 為 `nil` 或查無掉落表 → 不處理
2. 遍歷掉落表的 `entries`，對每個 `LootEntry` 擲骰判定 `dropRate`
3. 成功 → 隨機決定數量（`minQuantity`~`maxQuantity`）
4. 從 `ItemTemplateLoader` 查詢物品模板
5. 逐一加入背包：若背包已有相同物品且可堆疊且未滿 → `stackCount += 1`，否則建立新 `GameItem(from: template)`
6. 顯示掉落訊息（數量 > 1 時顯示 x 數量）

### 死亡處理

- 傳送至村莊（`03_01_village`，安全區）
- 恢復一半 HP（`maxHealth / 2`，最低 1）
- 恢復一半 SP（`maxStamina / 2`，最低 1）
- **不掉落物品、不損失經驗**

### 戰鬥中 UI 限制

- 「移動」按鈕 → 禁用（顯示「戰鬥中無法移動！」）
- 「攻擊」按鈕 → 禁用（顯示「你正在戰鬥中！」）
- 訊息顏色：傷害→紅色、閃避→薄荷色、勝利→紫色、掉落→黃色、升級→金色

## 二、數值平衡參考

### 角色 vs 怪物對照

以**戰士初始狀態**（HP 144 / 力量 14）持**鐵劍**（ATK 8）為例：

| 對手 | 角色攻擊力 | 怪物 DEF | 怪物 HP | 預估擊殺回合 | 怪物 ATK | 預估可承受回合 |
|------|:---------:|:--------:|:-------:|:-----------:|:--------:|:------------:|
| 雞 | 8 | 0 | 10 | ~2 回合 | 1 | ~144 回合 |
| 兔子 | 8 | 1 | 15 | ~3 回合 | 2 | ~72 回合 |
| 哥布林 | 8 | 2 | 25 | ~5 回合 | 5 | ~48 回合 |
| 野豬 | 8 | 4 | 40 | ~10 回合 | 8 | ~18 回合 |
| 灰狼 | 8 | 3 | 35 | ~7 回合 | 10 | ~14 回合 |
| 山賊 | 8 | 5 | 55 | ~19 回合 | 12 | ~72 回合 |

> 上表為簡化估算（ATK - DEF = 每回合傷害），實際傷害會受到 ±20% 隨機浮動影響。命中率、閃避率等因子未計入。

### 設計意圖

- **新手區**（兔子/雞）：任何職業裸裝即可輕鬆擊殺
- **過渡區**（哥布林）：稍有挑戰的人形怪物，初期金幣來源
- **進階區**（野豬/灰狼/毒蛙）：需要裝備與一定技能等級，毒蛙為沼澤入門
- **中高階區**（山賊/沼澤蜥蜴/礦坑蝙蝠）：穩定金幣與素材來源，蜥蜴 DEF 較高
- **高階區**（沼澤巨蟒/骷髏戰士/暗影刺客）：暗影刺客為「高攻低防」型，骷髏為「坦克」型
- **最高難度**（石像守衛）：ATK 18 / DEF 12 / HP 120，全遊戲最強，高金幣+稀有素材
- **職業限定裝備**（戰士巨劍/法師長袍/暗影之刃）：提供顯著戰力跳躍，作為等階 3 的目標獎勵
